## 4. Найденные дефекты

| ID | Заголовок | Приоритет | Тип | Статус |
|----|-----------|-----------|-----|--------|
| BUG-001 | Кнопка "галочка" не перемещает задачу в "Сделано" | Высокий | Баг приложения | Открыт |
| BUG-002 | Кнопка "галочка" не блокируется у выполненной задачи | Средний | Баг приложения (следствие BUG-001) | Открыт |
| BUG-003 | Автотесты не могут раскрыть блок "Комментарии" | Низкий | Баг тестов | Открыт |

### Подробности по дефектам

**BUG-001 (Высокий):** Функция `markTaskDone()` в `app.js` отправляет на сервер запрос без обязательного поля `title`. Сервер возвращает ошибку 400, задача не перемещается. Это основной способ отметки задачи выполненной — функциональность сломана.

**BUG-002 (Средний):** Является следствием BUG-001. После исправления BUG-001 этот баг, вероятно, будет закрыт автоматически.

**BUG-003 (Низкий):** Проблема в селекторах UI-тестов, а не в приложении. Вложенные `<details>` в HTML-шаблоне карточки путают Playwright. Ручное тестирование комментариев работает корректно.

Подробные баг-репорты: `bugs/bug-reports.md`

## 5. Рекомендации по улучшению

### Критические (до релиза)

- **Исправить BUG-001:** В функции `markTaskDone()` добавить передачу всех полей задачи на сервер (аналогично `moveDraggedTaskToStatus()`). Строка `app.js:218` — заменить `{ done: true, status: "done" }` на `{ ...task, done: true, status: "done" }`

### Желательные

- Добавить обработку ошибок при нажатии "галочка" — сейчас ошибка видна только в консоли, пользователь не понимает почему ничего не произошло
- Добавить валидацию дедлайна — приложение позволяет ставить дату в прошлом
- Добавить подтверждение при удалении одной задачи (сейчас удаляется без вопроса)

### Для тестов

- Исправить селекторы в UI-тестах для блока комментариев (BUG-003)
- Добавить тесты на граничные случаи (длинный текст, спецсимволы в названии)

## 6. Оценка готовности к релизу

**Оценка: Не готово к релизу**

Основная причина — BUG-001 (высокий приоритет). Кнопка "Выполнено" — одна из ключевых функций приложения. Пользователь не может отметить задачу выполненной основным способом (через кнопку "галочка"). Обходной путь существует (перетаскивание в колонку "Сделано" или смена статуса через редактирование), но это неочевидно для пользователя.

**После исправления BUG-001 приложение готово к релизу.** Остальные 96% тестов проходят успешно, все основные функции (авторизация, создание/редактирование/удаление задач, Drag & Drop, переключение видов, изоляция данных) работают корректно.
